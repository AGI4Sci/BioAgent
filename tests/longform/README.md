# SciForge Longform Evaluation Scripts

These scripts are the T060 manual regression suite for long multi-turn SciForge runs. They are intentionally separate from unit tests: a passing script means Codex operated the real UI, used browser evidence and Computer Use evidence, and recorded workspace/session refs.

## Running A Script

1. Start SciForge with `npm run dev` and open `http://localhost:5173/` in the in-app browser.
2. Pick one JSON file from `tests/longform/scenarios/`.
3. Prepare a run directory and pending manifest:

```sh
npm run longform:prepare -- --scenario <scenario-id> --workspace-path /path/to/workspace --backend <backend> --model-name <model>
```

4. Follow every `rounds[].prompt` and `rounds[].referenceOps` entry in order, using the generated `run-checklist.md`.
   To print the next runnable round and a matching recorder command:

```sh
npm run longform:next-round -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json
npm run longform:export-runbook -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json
```

5. Capture all required evidence listed in `evidencePlan`.
6. Record each completed round without hand-editing JSON:

```sh
npm run longform:record-round -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json --round <n> --status passed --observed "<what happened, including reference impact and blockers>" --artifact-ref <artifact-id> --execution-ref <execution-id> --screenshot-ref <screenshot-id>
```

7. Record top-level evidence without hand-editing JSON:

```sh
npm run longform:evidence-commands -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json
npm run longform:record-evidence -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json --kind artifact --id <artifact-id> --type <artifact-type> --path <artifact-path> --round <n> --status produced --summary "<how references changed the result>"
npm run longform:record-evidence -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json --kind execution-unit --id <execution-id> --status done --tool <tool-or-backend> --logRef <log-ref> --artifactRefs <artifact-id>
npm run longform:record-evidence -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json --kind screenshot --id <screenshot-id> --path <screenshot-path> --round <n> --caption "<browser or Computer Use evidence>"
```

8. Finalize status, scoring, notes, and any blockers without hand-editing JSON:

```sh
npm run longform:finalize -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json --status passed --coverage-stage real-data-success --score-task-completion <1-5> --score-reproducibility <1-5> --score-data-authenticity <1-5> --score-artifact-schema <1-5> --score-self-healing <1-5> --score-report-quality <1-5> --score-overall <1-5> --score-rationale "<why these scores are justified>" --notes "<how explicit references changed the result>"
```

For a blocked real run, use the same command with `--status repair-needed` plus `--failure-id`, `--failure-category`, `--failure-severity`, `--failure-summary`, and `--failure-repair-action`.

9. Run `npm run longform:validate` to check T060 evidence quality.
10. Check remaining evidence gaps before finalizing as passed:

```sh
npm run longform:missing-evidence -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json
npm run longform:evidence-commands -- --manifest docs/test-artifacts/deep-scenarios/<scenario-id>/manifest.json
```

11. Run `npm run longform:status` to review coverage across all six scripts and the current weekly regression count.
12. Run `npm run verify:deep` to validate scripts, manifests, status, finalization, and the generated deep report.

If the real backend, model quota, workspace server, or browser port is unavailable, still perform the UI reference smoke steps, record the blocker in the manifest, and leave the scenario status as `repair-needed` or `not-run`.

## Evidence Minimum

Every real run needs these evidence classes:

- `browser`: in-app browser screenshot or DOM observation.
- `computer-use`: Computer Use screenshot/coordinate operation record for selection, right-click, chip focus, or scrolling.
- `workspace`: session export, artifact ref, run ref, task code ref, log ref, or generated file path.

`not-run` manifests generated by `npm run longform:prepare` are allowed to pass validation as pending records. A `passed` longform manifest must include 6+ passed rounds, two or more reference operation types, browser evidence, Computer Use evidence, workspace refs, produced artifacts, and notes explaining how explicit references changed the conclusion, artifact, plan, or next step.

## Scoring

Use the dimensions embedded in each script:

- `goal-retention`
- `reference-use`
- `reasoning-continuity`
- `execution-capability`
- `failure-recovery`
- `artifact-quality`
- `reproducibility`
- `ui-usability`

Score each from 1 to 5 with `npm run longform:finalize`. A completed T060 regression should run at least two scripts per week and preserve session/workspace refs for later comparison.

For CI or a scheduled weekly regression, use:

```sh
npm run longform:weekly-plan -- --workspace-path /path/to/workspace --backend codex --model-name <model>
npm run longform:prepare-weekly -- --workspace-path /path/to/workspace --backend codex --model-name <model>
npm run longform:status -- --enforce-weekly
```

`longform:weekly-plan` selects the missing, pending, or blocked scenarios most useful for reaching the weekly target, skipping scenarios already passed with real data in the current ISO week.
`longform:prepare-weekly` creates pending manifests and checklists for the recommended scenarios. It skips scenarios that already have pending manifests unless `--force-pending` is supplied.
